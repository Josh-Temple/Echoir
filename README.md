# Echoir

Echoir is a fast, minimal listening-and-reconstruction app for intermediate+ English learners.

Core value: **reconstruction after listening**, not feature breadth.

## Learning modes

### Normal Mode (main mode)
Daily loop:
1. Listen to one sentence.
2. Reconstruct it mentally or out loud.
3. Reveal the answer.
4. Self-rate: **Good / Close / Missed**.
5. Continue quickly.

### Hard Mode (advanced)
One-sentence delay pattern:
1. Play sentence 1.
2. Play sentence 2.
3. Reconstruct sentence 1.
4. Reveal and self-rate.

Hard Mode is intentionally short and optional.

## Product constraints
- English-only learning flow.
- No Japanese in learning UX.
- No translation activity.
- No multiple-choice activity.
- Self-rating is required.
- Audio is manually managed (not generated by the app).
- Static site architecture (GitHub Pages compatible).

## Naming rules

### Sentence ID
Format:

```text
<work>-<level>-<serial>
```

Examples:
- `alice-a1-001`
- `alice-a2-014`
- `sherlock-b1-014`

### Audio filename
Format:

```text
<sentence_id>--n.mp3
```

Examples:
- `alice-a1-001--n.mp3`
- `sherlock-b1-014--n.mp3`

## Dataset format

Each dataset file in `src/data/` is a JSON array.

```json
{
  "id": "alice-a1-001",
  "unit": "alice",
  "level": "a1",
  "order": 1,
  "text": "She opened the door and looked outside.",
  "audio": "/audio/alice/alice-a1-001--n.mp3",
  "tags": ["past", "basic", "coordination"]
}
```

Field rules:
- `id` (required): stable unique key
- `unit` (required): content group name
- `level` (required): difficulty band
- `order` (required): sequence index within unit
- `text` (required): correct English sentence
- `audio` (required): path under `public/`
- `tags` (optional): label array

Do not add translation fields or Japanese helper fields.

## How to add a new dataset
1. Create or update `src/data/<work>.json`.
2. Assign stable IDs with `<work>-<level>-<serial>`.
3. Add matching audio files to `public/audio/<work>/`.
4. Use the exact static audio path in each item (`/audio/<work>/<sentence_id>--n.mp3`).
5. Keep IDs stable across edits.

## Deployment

### Vercel (recommended)
1. Import this repository into Vercel.
2. Build command: `npm run build`
3. Output directory: `dist`
4. Install command: `npm install`
5. Add environment variable only when needed:
   - `VITE_BASE_PATH` (optional, defaults to `/` in production builds)

This repo includes `vercel.json` with SPA rewrites so direct URL access falls back to `index.html`.

Note: Vite base-path configuration is read via `loadEnv` in `vite.config.ts`, so builds do not depend on Node global type declarations.

### GitHub Pages
1. Run `VITE_BASE_PATH=/Echoir/ npm run build`.
2. Publish `dist/` with your GitHub Pages workflow.
3. If repository name changes, set `VITE_BASE_PATH` to the new path.

## Review logic (MVP)
- `Good`: no urgent review item.
- `Close`: same-session repeat + later review interval.
- `Missed`: same-session repeat + later review interval.
- Later review intervals: **1 day, 3 days, 7 days**.

## Missing audio behavior
If audio cannot be played, the app shows:

> Audio unavailable for this item.

The session continues and the user can skip the item.
